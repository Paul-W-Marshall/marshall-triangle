# Contributing to Marshall Triangle

Thank you for your interest in contributing to the Marshall Triangle project.

## Licensing Notice

Before contributing, please understand our **tri-layer licensing model**:

| Component | License | Notes |
|-----------|---------|-------|
| **Source Code** | MIT License | Your code contributions will be MIT-licensed |
| **Visual Outputs** | CC BY-NC 4.0 | Generated images require attribution, no commercial use |
| **Conceptual Framework** | All Rights Reserved | The Marshall Triangle concept itself is not open-source |

## What You Can Contribute

- Bug fixes and performance improvements to the rendering engine
- UI/UX enhancements to the Streamlit application
- Documentation improvements
- Test coverage

## What Requires Authorization

The following require explicit permission from Fidelitas LLC:

- Modifications to the core conceptual framework
- Commercial use of generated visualizations
- Derivative works based on the Marshall Triangle methodology
- Use of the "Marshall Triangle" name in derivative projects

## How to Contribute

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request with a clear description

## Code Style

- Follow existing code conventions in `app.py` and `harmony_index.py`
- Include docstrings for new functions
- Maintain the tri-layer license headers in source files
